Software Information

Create Third Places is working on software will support the goal of promoting in person interaction through third places.

Guidelines for new software

Incremental change
Support making developmet accessible to more people

Create Third Places encourages support for third places through bottom-up action. To be consitent with this mission, software will be open source and minimize complexity so that more people can understand it. People will also be encouraged to modify the software for their own public space advocacy initiatives.

Software has clearly defined bounds on functionality
No use of LLMs, including chatbots
Minimize number of new features

While text or email communication features can be useful, they will not be supported by Create Third Places. People should be encouraged to meet new people in third places and then share contact information in person if they want to communicate online.

Another example of a feature that will not be included is the ability to rate third places. Ratings would take up space in the UI and be a distraction for some people. People are also unlikely to take time to rate every third place they visit, and there are already existing sites where people rate third places. For example, on December 23rd, 2025 there were 130 Yelp ratings for Dupont Circle in Washington DC, which is far less than the 1022 ratings on Google Maps. Google Maps reviews for DuPont Circle started being added more than 5 years ago, which would translate to 0 or 1 review posted on an average day and most people do not submit Google Maps ratings. DuPont Circle is an area of DC that receives a high amount of foot traffic, which averages to far more than 1 person per day.

While ratings can also be a way for people to communicate preferences for a third place, it can also lead to people rating a third place without having been there. Create Third Places encourages people to talk with people they met in person to get and share feedback on third places. Also, adding a ratings feature will increase complexity and increase the time necessary to test and maintain code.

See the tech strategy page for more information on how software should be developed.

Software made by Create Third Places

  • createthirdplaces.org website. Code contributions are welcome.
  • Decentralized event hosting: Create Third Places is currently working on designing the architecture and creating a proof of concept that will use the DMV Board Games website as a starting point.
  • Gatherspiel: A work in progress website for in person board game events. It will be an aggregator for the DMV Board Games website.
  • Places.js: Framework for building websites promoting in person interaction. It is currently used for the website DMV Board Games, a website for board game events in the DC area.

Create Third Places also relies on third party open source software. Here are the key open source software tools used by Create Third Places.

  • Docker
  • OpenJDK
  • PostgreSQL
  • Hosting software
    • Miraheze: Wiki hosting with Media Wiki
    • Source code hosting that uses Forgejo
