Contributing to Create Third Places

Create Third Places is looking for help in the following areas. If you are interested in contributing to one of these areas listed, or have other suggestions, email gulu@createthirdplaces.org. Also, Create Third Places has a strict no generative AI policy, and all content must be human created.

Programming
  • Javascript, HTML, and vanilla CSS on the frontend.
  • Java and PostgreSQL to build a proof of concept for a decentralized event hosting platform.
  • Moving Java backend away from Javalin. It is currently using an old version of Javalin, and using the latest Javalin version would go against the Create Third Places no generative AI policy.
  • Migrating DMV Board Games database away from Supabase to a self hosted Postgres instance. Recent additions of generative AI features conflict with the Create Third Places AI policy. Also, the self hosted instance can be an example that people can follow to create their own event hosting website.

Source code for Create Third Places is located here.

Website content and design
  • Pictures and descriptions of great public spaces that can be added to createthirdplaces.org.
  • Information about books related to third places.
  • Contribute to UI design

There is also a wiki at https://wiki.createthirdplaces.org that any human can edit. It uses an interface similar to Wikipedia.

Examples of content that would be relevant for wiki
  • Advice for hosting events in third places.
  • Advice on ways to help create or improve existing third places.
  • Biographies of third place advocates such as Ray Oldenburg.
  • Information about existing third places.
  • Links to online content related to third places.
